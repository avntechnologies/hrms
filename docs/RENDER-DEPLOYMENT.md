# Deploy PeopleFlow HRMS to Render

The repository includes a Render Blueprint at `render.yaml`. It creates:

- `peopleflow-hrms-api`: the .NET 10 Docker web service in Singapore.
- `peopleflow-hrms-web`: the Angular static site served through Render's CDN.

The Angular application calls `/api/v1` on its own origin. Render proxies `/api/*` to the API service, so tenant tokens and API URLs do not need to be compiled into the frontend.

## Before deploying

1. Push the `hrms` repository to GitHub or GitLab. `render.yaml` must be at the repository root.
2. Rotate the Neon database password before production if that credential has ever been pasted into a chat, ticket, log, or other non-secret channel.
3. Keep the API and Neon database in Singapore where possible to minimize latency.

## Create the Blueprint

1. In Render, select **New > Blueprint**.
2. Connect the repository containing this project.
3. Render detects `render.yaml` and asks for the values marked `sync: false`.
4. Enter the following secrets:

| Environment variable               | Value                                                                          |
| ---------------------------------- | ------------------------------------------------------------------------------ |
| `ConnectionStrings__Hrms`          | The Neon PostgreSQL URL or an Npgsql `Host=...;Database=...` connection string |
| `Bootstrap__PlatformAdminEmail`    | The initial platform owner email                                               |
| `Bootstrap__PlatformAdminPassword` | A unique password of at least 12 characters                                    |

`Jwt__SigningKey` is generated by Render and is never stored in Git. Do not reuse the bootstrap password as the JWT key.

5. Apply the Blueprint and wait for the API health check and static-site deployment to complete.

The default URLs are:

- Web: `https://peopleflow-hrms-web.onrender.com`
- API health: `https://peopleflow-hrms-api.onrender.com/health`

The database migration runs automatically when the API starts. Existing Neon data is retained. The bootstrap credentials create the platform administrator only when the platform tenant does not already exist; they do not reset an existing platform password.

## Service names and custom domains

The frontend API rewrite uses the default API hostname derived from `peopleflow-hrms-api`. If you rename that Render service, update the `/api/*` destination in `render.yaml` before deploying.

When adding a custom frontend domain, the same-origin `/api/*` proxy continues to work. If another browser application calls the API directly, add its exact HTTPS origin as another `Cors__AllowedOrigins__N` variable on the API service. Never use `*` together with credentialed CORS.

## Production recommendations

- The Blueprint uses Render's free API instance for an inexpensive first deployment. Free services can sleep and have cold starts. Move the API to `starter` or higher before onboarding paying customers.
- Use a paid Neon plan with backups, point-in-time recovery, alerts, and a connection limit suitable for the number of tenants.
- Configure Render deploy notifications, log streaming, uptime alerts, and a custom domain.
- Keep database migrations backward compatible for zero-downtime releases.
- Keep `Database__AutoMigrate=true` only while the API runs as a single instance. Before horizontal scaling, move migrations to a controlled pre-deploy job.
- Configure the customer's attendance-location retention and privacy policy before production use.

## Manual service configuration

If not using the Blueprint, use these settings:

### API

- Runtime: Docker
- Root directory: `backend`
- Dockerfile: `./Dockerfile`
- Region: Singapore
- Health check: `/health`
- Port: `10000`
- Add the environment variables declared for `peopleflow-hrms-api` in `render.yaml`.

### Frontend

- Runtime: Static
- Root directory: `frontend`
- Build command: `npm ci && npm run build`
- Publish directory: `dist/frontend/browser`
- Rewrite `/api/*` to `https://peopleflow-hrms-api.onrender.com/api/*`.
- Rewrite `/*` to `/index.html` after the API rule.
